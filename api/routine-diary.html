API 레퍼런스

루틴 & 일기

모든 엔드포인트에 인증 필요 (Authorization: Bearer <accessToken>).

루틴

사용자당 AM 루틴 1개, PM 루틴 1개만 가질 수 있습니다 (DB UNIQUE 제약: user_id + routine_period).

GET/api/routine?period=AM|PM

현재 루틴 조회. 등록된 루틴이 없으면 404 반환 — 프론트에서 null 처리해야 합니다.

// Response data (루틴이 있을 때)
{
  "id": 10,
  "routinePeriod": "AM",
  "products": [
    {
      "productId": 1,
      "productName": "레티놀 세럼",
      "brand": "BRAND A",
      "imageUrl": "https://...",
      "stepOrder": 1
    },
    {
      "productId": 5,
      "productName": "수분 크림",
      "brand": "BRAND B",
      "imageUrl": "https://...",
      "stepOrder": 2
    }
  ]
}
stepOrder가 순서 필드입니다 (order 아님). 프론트 타입 RoutineItem의 해당 필드명을 확인하세요.

PUT/api/routine

루틴 upsert. 루틴이 없으면 생성, 있으면 products 전체 교체.

// Request body
{
  "routinePeriod": "AM",
  "products": [
    { "productId": 1, "stepOrder": 1 },
    { "productId": 5, "stepOrder": 2 }
  ]
}

DELETE/api/routine?period=AM|PM

해당 period의 루틴 전체 삭제.

일기

사용자당 날짜별 1개만 허용됩니다 (DB UNIQUE 제약: user_id + log_date).

GET/api/diary?year=&month=

월별 일기 목록 조회.

GET /api/diary?year=2026&month=6
// Response data (array)
[
  {
    "logId": 1,
    "logDate": "2026-06-10",
    "skinStatus": "GOOD",
    "keywords": ["보습", "트러블없음"],
    "memo": "오늘 피부 컨디션 최고",
    "photoUrl": null
  }
]
필드명: logDate, skinStatus, memo — date, mood, note가 아닙니다. 프론트 타입 DiaryEntry와 맞춰야 합니다.

POST/api/diary

일기 작성.

// Request body
{
  "logDate": "2026-06-11",
  "skinStatus": "NORMAL",
  "keywords": ["보습", "약간건조"],
  "memo": "오전에 세럼 추가함",
  "photoUrl": null
}

GET/api/diary/:logId

일기 단건 조회.

PUT/api/diary/:logId

일기 수정. Request body는 POST와 동일.

DELETE/api/diary/:logId

일기 삭제.
