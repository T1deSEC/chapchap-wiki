API 레퍼런스

인증 & 사용자

공개 경로: /api/auth/**

나머지 전체: Authorization: Bearer <accessToken> 필요. 미인증 시 401.

모든 응답은 { "success": true, "data": <T>, "message": null } 래퍼. 프론트 axios interceptor가 자동 언래핑하므로 훅에서 .data가 바로 실제 페이로드입니다.

인증

POST/api/auth/register

회원가입. 이메일 중복 시 409. IP당 토큰 불일치 10회 누적 시 1시간 차단(429).

// Request body
{
  "name": "홍길동",
  "email": "user@example.com",
  "password": "securePassword123!",
  "skinType": "DRY",
  "registrationToken": "<REGISTRATION_SECRET 값>"
}
⚠ registrationToken 필수 — 서버 환경변수 REGISTRATION_SECRET과 일치해야 합니다. 비밀번호는 최소 8자. 
// Response data
{
  "accessToken": "eyJhbGci...",
  "refreshToken": "eyJhbGci...",
  "user": { "id": 1, "name": "홍길동", "email": "user@example.com" }
}

POST/api/auth/login

로그인.

// Request body
{ "email": "user@example.com", "password": "securePassword123!" }
// Response data
{
  "accessToken": "eyJhbGci...",
  "refreshToken": "eyJhbGci...",
  "user": { "id": 1, "name": "홍길동", "email": "user@example.com", "skinType": "DRY" }
}

POST/api/auth/refresh

Access Token 재발급. Refresh Token이 만료되었거나 DB에 없으면 401.

// Request body
{ "refreshToken": "eyJhbGci..." }

// Response data
{ "accessToken": "eyJhbGci...(new)" }

POST/api/auth/logout

로그아웃. 서버에서 Refresh Token을 DB에서 폐기합니다. 인증 필요.

// Response data: null (success: true만 반환)

사용자

GET/api/users/me

내 프로필 조회.

// Response data
{
  "id": 1,
  "name": "홍길동",
  "email": "user@example.com",
  "nickname": "길동이",
  "skinType": "DRY",
  "skinConcerns": ["ACNE", "SENSITIVITY"],
  "birthYear": 1995,
  "gender": "MALE"
}

PUT/api/users/me/skin-profile

피부 타입 및 기본 정보 업데이트.

// Request body
{ "skinType": "COMBINATION", "gender": "FEMALE", "birthYear": 1998 }

PUT/api/users/me/skin-concerns

피부 고민 목록 교체 (전체 덮어쓰기).

// Request body
{ "concerns": ["DRYNESS", "PIGMENTATION"] }

GET / POST / DELETE/api/users/me/ingredient-preferences

선호 성분 목록 조회·추가·삭제.

// POST Request body
{ "ingredientId": 42, "preferenceType": "PREFER" }  // PREFER | AVOID

// DELETE: /api/users/me/ingredient-preferences/:ingredientId

PUT/api/users/me/nickname

{ "nickname": "새닉네임" }

PUT/api/users/me/password

{ "currentPassword": "old123!", "newPassword": "new456!" }

GET / PUT/api/users/me/notification-settings

알림 카테고리별 ON/OFF 조회·수정. 3개 카테고리 (루틴 conflict/caution은 단일 routineAnalysisEnabled 항목).

// GET Response data
{
  "ingredientAnalysisEnabled": true,
  "ingredientRecommendEnabled": true,
  "routineAnalysisEnabled": true
}

// PUT Request body — 변경할 항목만 포함
{ "routineAnalysisEnabled": false }
