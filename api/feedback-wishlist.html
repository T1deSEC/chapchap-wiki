피드백 & 찜

모든 엔드포인트에 인증 필요.

피드백

피드백은 제품당 1개입니다. POST는 upsert — 이미 있으면 수정, 없으면 생성.

POST/api/feedback

피드백 등록 또는 수정 (upsert).

// Request body
{
  "productId": 1,
  "reaction": "good",
  "memo": "피부에 잘 맞았어요",
  "rating": 4,
  "usagePeriod": "1개월"
}
필드 타입 설명
productIdnumber필수
reactionstring필수. good | neutral | trouble
memostring선택
ratingnumber 1~5선택 (V15 추가)
usagePeriodstring선택 (V15 추가). 예: "1개월"
reaction → effective 파생 규칙
good → effective = true
neutral / trouble → effective = false
응답의 FeedbackResponse에서 effective 필드로 확인 (isEffective 아님). 
// Response data
{
  "productId": 1,
  "productName": "레티놀 세럼",
  "brand": "BRAND A",
  "imageUrl": "https://...",
  "reaction": "good",
  "effective": true,
  "memo": "피부에 잘 맞았어요",
  "rating": 4,
  "usagePeriod": "1개월",
  "createdAt": "2026-06-11T10:00:00"
}

GET/api/feedback

내 피드백 목록 조회. 응답은 FeedbackResponse[] 배열.

DELETE/api/feedback/:productId

특정 제품 피드백 삭제.

찜 (Wishlist)

GET/api/wishlist

찜 목록 조회.

// Response data (array)
[
  {
    "productId": 1,
    "productName": "레티놀 세럼",
    "brand": "BRAND A",
    "imageUrl": "https://...",
    "category": "SERUM",
    "addedAt": "2026-06-10T09:00:00"
  }
]
⚠ 단일 id 필드 없음 — WishlistItemResponse에는 id가 없습니다. 리스트 렌더링 key와 삭제 시 모두 productId를 사용하세요.

POST/api/wishlist/:productId

찜 추가. 이미 찜한 제품이면 409 반환.

DELETE/api/wishlist/:productId

찜 삭제.
