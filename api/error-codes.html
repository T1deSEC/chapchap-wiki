에러 코드

BusinessException(ErrorCode, HttpStatus) 기반 에러 응답 레퍼런스

에러 응답 형식

모든 에러 응답은 공통 래퍼 ApiResponse를 사용합니다.

{
  "success": false,
  "data": null,
  "message": "에러 메시지 (ErrorCode.message)"
}

에러 메시지는 ErrorCode 열거형에 정의된 한국어 문자열이 그대로 내려갑니다. GlobalExceptionHandler가 BusinessException을 캐치해 HTTP 상태 코드와 함께 이 형식으로 반환합니다.

에러 코드 전체 목록

ErrorCode HTTP 메시지 발생 상황
INVALID_INPUT 400 / 503 ★ 잘못된 입력입니다 피부 프로필 미설정(400), AI 서비스 호출 실패(503)
RESOURCE_NOT_FOUND 404 리소스를 찾을 수 없습니다 범용 리소스 미발견 (잘 사용되지 않음)
INVALID_CREDENTIALS 401 이메일 또는 비밀번호가 잘못되었습니다 로그인 실패, 비밀번호 변경 시 현재 비밀번호 불일치
DUPLICATE_EMAIL 409 이미 사용 중인 이메일입니다 회원가입 시 이미 존재하는 이메일
INVALID_TOKEN 401 유효하지 않은 토큰입니다 JWT 서명·형식 오류, 리프레시 토큰 미발견
EXPIRED_TOKEN 401 만료된 토큰입니다 액세스 토큰 만료 (30분), 리프레시 토큰 만료 (14일)
USER_NOT_FOUND 404 사용자를 찾을 수 없습니다 JWT subject userId에 해당하는 유저 DB에 없음
PRODUCT_NOT_FOUND 404 제품을 찾을 수 없습니다 존재하지 않는 productId로 조회·루틴 추가·피드백 작성
INGREDIENT_NOT_FOUND 404 성분을 찾을 수 없습니다 성분 선호도 삭제 시 해당 ingredientId 미발견
DIARY_NOT_FOUND 404 일기를 찾을 수 없습니다 해당 날짜 일기 미발견 (조회·수정·삭제)
DIARY_ALREADY_EXISTS 409 해당 날짜에 이미 일기가 존재합니다 같은 날짜에 일기 중복 생성 시도
DIARY_ACCESS_DENIED — 일기에 접근할 권한이 없습니다 정의됨, 현재 미사용 ★★
ROUTINE_NOT_FOUND 404 루틴을 찾을 수 없습니다 해당 period 루틴 미발견 (조회·수정·삭제)
ROUTINE_ACCESS_DENIED — 루틴에 접근할 권한이 없습니다 정의됨, 현재 미사용 ★★
INVALID_REGISTRATION_TOKEN 403 유효하지 않은 가입 코드입니다 회원가입 시 registrationToken이 REGISTRATION_SECRET과 불일치
REGISTRATION_TOO_MANY_ATTEMPTS 429 가입 시도 횟수가 초과되었습니다 동일 IP에서 10회 가입 실패 시 1시간 차단

특수 케이스

★ INVALID_INPUT — HTTP 상태 코드가 두 가지

BusinessException은 ErrorCode와 HttpStatus를 분리해 받습니다. INVALID_INPUT은 동일한 에러 코드가 상황에 따라 다른 HTTP 상태 코드로 내려갑니다.

상황 HTTP throw 위치
AI 분석 요청 시 피부 프로필 미설정 400 Bad Request AiAnalysisService
Python AI 서비스 호출 실패 (네트워크·타임아웃) 503 Service Unavailable AiAnalysisService 
// AiAnalysisService.java
throw new BusinessException(ErrorCode.INVALID_INPUT, HttpStatus.BAD_REQUEST);   // 400
throw new BusinessException(ErrorCode.INVALID_INPUT, HttpStatus.SERVICE_UNAVAILABLE); // 503
클라이언트가 message: "잘못된 입력입니다"를 받았다면 HTTP 상태 코드로 원인을 구분합니다. 400은 요청 수정이 필요하고, 503은 서버 측 문제로 재시도 대상입니다.

★★ DIARY_ACCESS_DENIED / ROUTINE_ACCESS_DENIED — 미사용

ErrorCode 열거형에는 정의되어 있지만 현재 어떤 서비스에서도 throw하지 않습니다. 일기·루틴은 현재 소유자 확인 없이 userId 기준 조회만 수행합니다. 향후 공유 기능 추가 시 사용할 예정입니다.

구현 구조

// BusinessException.java
public class BusinessException extends RuntimeException {
    private final ErrorCode errorCode;
    private final HttpStatus httpStatus;

    public BusinessException(ErrorCode errorCode, HttpStatus httpStatus) {
        super(errorCode.getMessage());
        this.errorCode = errorCode;
        this.httpStatus = httpStatus;
    }
}
// GlobalExceptionHandler.java
@ExceptionHandler(BusinessException.class)
public ResponseEntity<ApiResponse<Void>> handleBusinessException(BusinessException ex) {
    return ResponseEntity
        .status(ex.getHttpStatus())
        .body(ApiResponse.error(ex.getMessage()));
}
대부분의 에러 코드는 고정 HTTP 상태 코드와 함께 throw됩니다. 예: throw new BusinessException(ErrorCode.USER_NOT_FOUND, HttpStatus.NOT_FOUND). 각 서비스 클래스에서 적절한 HttpStatus를 직접 지정합니다.

Spring Security 에러 (BusinessException 외)

인증·인가 실패는 BusinessException을 거치지 않고 Spring Security가 직접 처리합니다.

상황 HTTP 처리 주체
Authorization 헤더 없음 / JWT 검증 실패 401 JwtAuthenticationFilter → authenticationEntryPoint
인증은 됐으나 권한 부족 403 accessDeniedHandler
SecurityConfig에 authenticationEntryPoint가 명시적으로 설정되어 있어 인증 실패 시 403이 아닌 401이 반환됩니다. 미인증 상태에서 보호된 API를 호출하면 항상 401을 기대합니다.
