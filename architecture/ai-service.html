AI 서비스

FastAPI + LangGraph 구조, 성분·루틴 분석 플로우, Spring Boot 연동.

서비스 개요

Python FastAPI 서버(:8000)로 분리된 AI 마이크로서비스. Spring Boot가 WebClient로 HTTP 요청을 보내며, LangGraph와 OpenAI gpt-4o-mini를 사용해 성분 분석·루틴 진단을 수행한다.

메서드 경로 설명
POST/analyze/ingredient제품 성분 AI 분석 (LangGraph StateGraph)
POST/analyze/routineAM/PM 루틴 갈등 진단 (ReAct agent)
POST/recommend/ingredients성분 추천
GET/health헬스체크

성분 분석 플로우

입력: 제품 성분 목록 + 사용자 피부 프로필
  └─► collect_and_triage          (성분 수집·분류)
        └─► analyze_one_ingredient  (병렬, 성분별 GPT 호출)
              └─► aggregate_score   (안전도 점수 가중 평균 계산)
                    └─► generate_summary (최종 요약 생성)

출력: IngredientAnalysis[] + safety_score(0-100)

각 성분은 11개 피부 영향 카테고리로 평가된다.

루틴 분석 플로우

입력: AM/PM 루틴 제품 목록 + 사용자 피부 프로필
  └─► ReAct agent (create_react_agent)
        ├─► Tool: get_product_ingredients      (제품 성분 조회)
        └─► Tool: find_potentially_conflicting_ingredients (성분 충돌 탐지)

출력: status (ok / conflict / caution)
분석 결과 상태: ok — 충돌 없음, conflict — 성분 충돌 감지 (ROUTINE_CONFLICT 알림 발송), caution — 주의 필요 (ROUTINE_CAUTION 알림 발송)

Spring Boot ↔ Python 통신

React 앱
  └─► POST /api/analysis/ingredient  (Spring Boot :8080)
        └─► AiAnalysisService (WebClient)
              └─► POST /analyze/ingredient  (Python FastAPI :8000)
                    └─► LangGraph + OpenAI gpt-4o-mini
  • 요청/응답: JSON (camelCase — Pydantic alias_generator=to_camel)
  • Spring: WebClient 비동기 HTTP 클라이언트
  • Python: FastAPI + Pydantic, response_model_by_alias=True
  • 오류 시: Spring이 BusinessException으로 래핑하여 클라이언트에 전달
Python 모델에 ConfigDict(alias_generator=to_camel, populate_by_name=True)와 엔드포인트에 response_model_by_alias=True가 없으면 422 Unprocessable Entity가 발생한다.

skin_impacts 저장 구조

user_product_ai_analysis 테이블의 skin_impacts 컬럼은 JSONB 타입으로, 11개 카테고리 점수를 JSON으로 저장한다.

JPA 엔티티에서 @JdbcTypeCode(SqlTypes.JSON) + String 타입으로 선언. Jackson이 직렬화/역직렬화를 담당한다.
