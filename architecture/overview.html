아키텍처 개요

CHAPCHAP 시스템의 전체 구조, 기술 스택, 핵심 설계 원칙.

전체 레이어 다이어그램

Browser (React + Vite)
  └─► Vite Dev Server (:5173)
        └─► /api/* 프록시
              └─► Spring Boot (:8080)
                    ├─► PostgreSQL (:5432)
                    └─► Python AI Service (:8000)
                              └─► OpenAI API (gpt-4o-mini)

기술 스택

레이어 기술
프론트엔드React 18 + TypeScript + Vite, TailwindCSS, TanStack Query v5, Zustand, React Router v6, Axios, Framer Motion
백엔드Spring Boot 3.3.4, Java 22, Gradle, Spring Security 6, jjwt 0.12.6, Spring Data JPA, Flyway
데이터베이스PostgreSQL 16, Flyway 마이그레이션 V1–V15
AI 서비스Python 3.11, FastAPI, LangGraph, LangChain-OpenAI, gpt-4o-mini
인프라Docker, Docker Compose (3 services: db / api / ai)

핵심 설계 원칙

ApiResponse 래퍼: 모든 백엔드 응답은 { success, data, message } 래퍼로 감싼다. 프론트 axios 인터셉터(unwrapApiResponse)가 자동으로 벗겨내므로 훅에서 r.data는 이미 실제 페이로드다.
JWT 토큰 수명: Access Token 30분, Refresh Token 14일. 401 수신 시 프론트 인터셉터가 자동으로 갱신을 시도하고 원본 요청을 재시도한다.
DB 마이그레이션: Flyway로 관리. 현재 V15까지 적용됨. 다음 마이그레이션은 V16부터 작성.
TanStack Query: staleTime: 5분, retry: 1 전역 기본값. 페이지네이션 응답의 실제 배열은 .content에 있다.
