Flyway 마이그레이션 이력

버전 내용 비고
V1 유저 도메인 8개 테이블 (users, user_skin_profile, user_skin_concern, user_ingredient_preference, user_routine, user_routine_product, user_daily_skin_log, user_daily_skin_log_keyword) gender/birth_year nullable
V2 products, ingredients, product_ingredients
V3 refresh_tokens
V4 user_wishlist, product_feedback
V5 시드 데이터 — 성분 10개, 제품 10개 (샘플) 데이터 전용
V6 product_feedback.reaction VARCHAR(30) 추가
V7 notifications 테이블
V8 user_ingredient_recommendation, user_recommended_ingredient, user_avoid_ingredient, user_recommended_product
V9 user_notification_settings (카테고리별 알림 on/off)
V10 user_routine에 UNIQUE(user_id, routine_period) 추가
V11 user_product_ai_analysis 테이블 (skin_impacts JSONB)
V12 home 관련 테이블
V13 시드 데이터 추가 데이터 전용
V14 시드 데이터 추가 데이터 전용
V15 product_feedback에 rating, usage_period 컬럼 추가
다음 마이그레이션: V16부터 시작. V5·V13·V14는 스키마 변경 없는 시드 데이터 파일이다.

주요 테이블

users

컬럼 타입 설명
idBIGINTPK
emailVARCHARUNIQUE NOT NULL
passwordVARCHARNOT NULL (BCrypt 해시)
nicknameVARCHARNOT NULL
created_atTIMESTAMP가입 일시

user_skin_profile

컬럼 타입 설명
user_idBIGINTPK, FK → users.id
skin_typeVARCHARNOT NULL (건성/지성/복합성/민감성 등)
genderVARCHARnullable
birth_yearINTnullable

products

컬럼 타입 설명
idBIGINTPK
nameVARCHARNOT NULL
brandVARCHAR브랜드명
categoryVARCHAR제품 카테고리
image_urlVARCHAR제품 이미지 URL

ingredients

컬럼 타입 설명
idBIGINTPK
inci_nameVARCHARNOT NULL (INCI 국제 성분명)
ko_nameVARCHAR한국어 성분명
functionVARCHAR성분 기능 (보습, 방부 등)
ewg_gradeVARCHAREWG 안전 등급

product_ingredients

컬럼 타입 설명
product_idBIGINT복합 PK, FK → products.id
ingredient_idBIGINT복합 PK, FK → ingredients.id
concentration_orderINT전성분 표기 순서 (낮을수록 고함량)

user_routine

컬럼 타입 설명
idBIGINTPK
user_idBIGINTFK → users.id
routine_periodVARCHARAM 또는 PM
record_dateDATE루틴 기록 날짜
UNIQUE(user_id, routine_period) — 사용자당 AM/PM 각 1개

user_routine_product

컬럼 타입 설명
routine_idBIGINTFK → user_routine.id
product_idBIGINTFK → products.id
step_orderINT루틴 내 제품 적용 순서

user_daily_skin_log

컬럼 타입 설명
idBIGINTPK
user_idBIGINTFK → users.id
log_dateDATENOT NULL, 기록 날짜
skin_statusVARCHAR당일 피부 상태
memoTEXT자유 메모
UNIQUE(user_id, log_date) — 날짜당 1개

product_feedback

컬럼 타입 설명
idBIGINTPK
user_idBIGINTFK → users.id
product_idBIGINTFK → products.id
reactionVARCHARgood / neutral / trouble (V6 추가)
ratingINT평점 (V15 추가)
usage_periodVARCHAR사용 기간 (V15 추가)

user_wishlist

컬럼 타입 설명
user_idBIGINT복합 PK, FK → users.id
product_idBIGINT복합 PK, FK → products.id
added_atTIMESTAMP찜 등록 일시

notifications

컬럼 타입 설명
idBIGINTPK
user_idBIGINTFK → users.id
typeVARCHAR알림 유형
messageTEXT알림 본문
is_readBOOLEAN읽음 여부
created_atTIMESTAMP생성 일시

refresh_tokens

컬럼 타입 설명
idBIGINTPK
user_idBIGINTFK → users.id
tokenVARCHARUNIQUE, Refresh Token 값
expires_atTIMESTAMP만료 일시
is_revokedBOOLEAN폐기 여부

user_product_ai_analysis

컬럼 타입 설명
idBIGINTPK
user_idBIGINTFK → users.id
product_idBIGINTFK → products.id
skin_impactsJSONB11개 피부 카테고리별 영향 점수
analyzed_atTIMESTAMPAI 분석 수행 일시

테이블 관계 (ERD)

users
  ├── user_skin_profile        (1:1)
  ├── user_skin_concern        (1:N)
  ├── user_ingredient_preference (1:N)
  ├── user_daily_skin_log      (1:N, UNIQUE log_date)
  ├── user_routine             (1:2, UNIQUE routine_period AM/PM)
  │     └── user_routine_product (1:N, step_order)
  ├── user_wishlist            (N:M → products)
  ├── product_feedback         (1:N → products, reaction/rating)
  ├── notifications            (1:N)
  ├── refresh_tokens           (1:N)
  └── user_product_ai_analysis (1:N → products, JSONB skin_impacts)

products
  └── product_ingredients      (N:M → ingredients, concentration_order)
