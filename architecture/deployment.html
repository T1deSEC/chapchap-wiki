배포

로컬 개발 환경 및 EC2 단일 서버 프로덕션 배포 가이드.

로컬 개발 환경

Docker Compose 서비스 구성

서비스 이미지/빌드 포트 프로파일 설명
db postgres:16 5432 (항상) PostgreSQL 16 데이터베이스
api build: . 8080 full Spring Boot 애플리케이션
ai build: ./chapchap-ai 8000 full Python FastAPI AI 서비스
--profile full 없이 실행하면 db만 기동됩니다. api·ai는 full 프로파일에만 포함됩니다.

시작 명령어

# DB만 기동 (일반 개발)
cd CHAPCHAP-Backend
docker-compose up db -d
.\gradlew.bat bootRun

# 전체 스택 (AI 포함)
.\gradlew.bat bootJar
docker-compose --profile full up --build -d

# 프론트엔드
cd CHAPCHAP-Frontend
npm run dev    # http://localhost:5173 → /api/* 를 :8080으로 프록시
bootJar 없이 docker-compose --profile full up --build를 실행하면 api 컨테이너가 이전 JAR로 빌드됩니다. 항상 bootJar 먼저 실행하십시오.

서비스 간 통신 (로컬)

Vite Dev Server (:5173)
  └─► /api/* 프록시 → Spring Boot (:8080)
        ├─► PostgreSQL (:5432) — JDBC
        └─► Python AI (:8000) — HTTP (WebClient)
              └─► OpenAI API — HTTPS

프로덕션 배포 (EC2 단일 서버)

배포 구조

브라우저 → http://EC2_IP (nginx :80)
              ├── /        → 프론트 정적 파일 (빌드된 React)
              └── /api/*   → Spring Boot :8080 (내부 프록시)

EC2 Docker 컨테이너:
  ├─ nginx   :80    (프론트 서빙 + API 프록시)
  ├─ api     :8080  (Spring Boot)
  ├─ db      :5432  (PostgreSQL)
  └─ ai      :8000  (Python FastAPI)
브라우저 기준 프론트와 API가 같은 origin(http://EC2_IP)이므로 CORS 설정과 Mixed Content 문제가 발생하지 않습니다.

EC2 인스턴스 설정

항목 값
AMIUbuntu Server 22.04 LTS
Instance typet2.micro (프리 티어)
Storage20 GiB gp3
인바운드 포트22 (SSH, 내 IP), 80 (nginx, 0.0.0.0/0)
탄력적 IP할당 권장 — 재시작 후에도 IP 고정

EC2 초기 환경 구성

# Swap 2GB (1GB RAM으로 컨테이너 4개 운용에 필수)
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile && sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# Docker 설치 후 재접속
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker ubuntu
# exit 후 재접속

# JDK 22 (Spring Boot JAR 빌드용)
curl -s "https://get.sdkman.io" | bash
source ~/.sdkman/bin/sdkman-init.sh
sdk install java 22.0.2-tem

# Node.js 20 (프론트엔드 빌드용)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs git
Swap 설정을 생략하면 컨테이너 4개(nginx + api + db + ai)가 동시에 올라오는 시점에 OOM으로 프로세스가 강제 종료됩니다.

프론트엔드 빌드

cd ~
git clone https://github.com/<username>/CHAPCHAP-Frontend.git
cd CHAPCHAP-Frontend
npm ci
npm run build

백엔드 배포 및 전체 스택 실행

cd ~
git clone https://github.com/<username>/CHAPCHAP-Backend.git
cd CHAPCHAP-Backend

# 프론트엔드 빌드 결과물 복사 — 백엔드 클론 후에 실행해야 함
mkdir -p frontend-dist
cp -r ~/CHAPCHAP-Frontend/dist/* frontend-dist/

# .env 작성 (gitignore됨 — EC2에서 직접 생성, .env.example 참고)
cat > .env << 'EOF'
JWT_SECRET=<32자 이상 임의 문자열>
OPENAI_API_KEY=<키>
OPENWEATHERMAP_API_KEY=<키>
REGISTRATION_SECRET=<가입 코드>
CORS_ALLOWED_ORIGINS=http://localhost:5173
EOF

# JAR 빌드
chmod +x gradlew
./gradlew bootJar --no-daemon

# 전체 스택 기동 (db + api + ai + nginx)
docker compose --profile full up --build -d

# 확인
docker compose ps
docker compose logs -f api   # "Started ChapchapApplication" 확인

환경 변수 전체 목록

.env 파일에 정의합니다 (gitignore됨). docker-compose.yml이 자동으로 로드합니다.

변수 대상 기본값 설명
JWT_SECRET api dev 기본값 JWT 서명 키 (32자 이상 필수)
OPENAI_API_KEY ai 없음 OpenAI API 키
OPENWEATHERMAP_API_KEY api dev-placeholder OpenWeatherMap API 키
REGISTRATION_SECRET api local-dev-secret 회원가입 가입 코드. 프로덕션에서는 반드시 강한 값으로 교체
CORS_ALLOWED_ORIGINS api http://localhost:5173 쉼표 구분 허용 Origin 목록
EC2 nginx 단일 서버 구성에서는 프론트와 API가 same-origin이므로 브라우저가 CORS 요청을 발생시키지 않습니다. CORS_ALLOWED_ORIGINS는 실질적으로 동작에 영향을 주지 않지만, 로컬 개발 시에는 http://localhost:5173이 필요합니다. 값을 변경한 뒤에는 docker compose restart api만 실행하면 됩니다 (전체 재빌드 불필요).

재배포 방법

백엔드 코드 변경 시

cd CHAPCHAP-Backend
git pull
./gradlew bootJar --no-daemon
docker compose --profile full up --build -d

환경변수만 변경 시

# .env 수정 후
docker compose restart api

프론트엔드 코드 변경 시

cd ~/CHAPCHAP-Frontend
git pull
npm ci
npm run build
rm -rf ~/CHAPCHAP-Backend/frontend-dist/*
cp -r dist/* ~/CHAPCHAP-Backend/frontend-dist/
docker compose -f ~/CHAPCHAP-Backend/docker-compose.yml restart nginx

로그 확인

docker compose logs -f api          # Spring 실시간 로그
docker compose logs --tail=100 ai   # AI 최근 100줄
docker compose ps                   # 컨테이너 상태
