Docker Compose 서비스 구성, 시작 명령어, 포트 맵, 환경 변수.

Docker Compose 서비스 구성

서비스 이미지/빌드 포트 프로파일 설명
db postgres:16 5432 (항상) PostgreSQL 16 데이터베이스
api build: . 8080 full Spring Boot 애플리케이션
ai build: ./chapchap-ai 8000 full Python FastAPI AI 서비스
--profile full 없이 docker-compose up을 실행하면 db만 기동된다. api·ai는 full 프로파일에만 포함된다.

시작 명령어

# 개발 (DB만)
cd CHAPCHAP-Backend
docker-compose up db -d

# Spring Boot 로컬 실행
.\gradlew.bat bootRun

# 전체 스택 (AI 포함)
.\gradlew.bat bootJar                              # 먼저 JAR 빌드
docker-compose --profile full up --build -d        # 전체 기동

# 프론트엔드
cd CHAPCHAP-Frontend
npm run dev    # http://localhost:5173 → /api/* 를 :8080으로 프록시

서비스 기동 순서

1. .\gradlew.bat bootJar     — 실행 가능한 JAR 생성
2. docker-compose --profile full up --build -d
   ├─► db 서비스 기동 (PostgreSQL)
   ├─► api 서비스 빌드·기동 (JAR 복사, JRE 이미지)
   └─► ai 서비스 빌드·기동 (Python 이미지)
bootJar 없이 docker-compose --profile full up --build를 실행하면 api 컨테이너가 오래된 JAR로 빌드된다. 항상 bootJar 먼저.

포트 맵

서비스 컨테이너 포트 호스트 포트 설명
PostgreSQL 5432 5432 DB 직접 접속 (개발용)
Spring Boot 8080 8080 REST API
Python AI 8000 8000 FastAPI AI 서비스
Vite Dev 5173 5173 프론트엔드 개발 서버 (컨테이너 아님)

환경 변수

.env 파일에 정의 (gitignore됨). docker-compose.yml이 자동으로 로드한다.

변수 대상 서비스 설명
JWT_SECRET Spring Boot (api) JWT 서명 키 (32자 이상)
OPENAI_API_KEY Python AI (ai) OpenAI API 키
OPENWEATHERMAP_API_KEY Spring Boot (api) OpenWeatherMap API 키
.env 파일은 절대 커밋하지 않는다. .gitignore에 등록되어 있으며, 새 환경 설정 시 직접 생성해야 한다.

서비스 간 통신

Vite Dev Server (:5173)
  └─► /api/* 프록시 → Spring Boot (:8080)
        ├─► PostgreSQL (:5432) — JDBC
        └─► Python AI (:8000) — HTTP (WebClient)
              └─► OpenAI API — HTTPS
