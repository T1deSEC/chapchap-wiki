  • React 18 + TypeScript + Vite

// 401 시 자동 토큰 갱신 후 원본 요청 재시도
interceptor.onError(async (error) => {
  if (error.response?.status === 401) {
    await refreshToken();
    return retry(error.config);
  }
});

DB 스키마 (Flyway V1–V15)

버전 내용
V1users, user_skin_profile, user_skin_concern, user_ingredient_preference, user_routine, user_routine_product, user_daily_skin_log, user_daily_skin_log_keyword
V2products, ingredients (inci_name, ko_name, description), product_ingredients (concentration_order)
V3refresh_tokens (token, user_id, expires_at)
V4user_wishlist, product_feedback (is_effective, memo)
V6product_feedback에 reaction VARCHAR(30) 추가
V7notifications (type, title, body, is_read)
V8user_ingredient_recommendation, user_recommended_ingredient, user_avoid_ingredient, user_recommended_product
V9user_notification_settings (카테고리별 ON/OFF)

V11ingredients.safety_level DROP, user_product_ai_analysis·user_product_ai_ingredient 신규 생성
V12ingredients.function_tags DROP, user_product_ai_analysis에 skin_impacts JSONB 추가
V15product_feedback에 rating SMALLINT, usage_period VARCHAR(20) 추가
V5·V13·V14는 시드/테스트 데이터 마이그레이션으로 스키마 변경 없음. 다음 마이그레이션은 V16부터.

JWT 인증 흐름

1. POST /api/auth/login
   └─► 서버: Access Token (30분) + Refresh Token (14일) 발급

2. 이후 모든 요청
   └─► Header: Authorization: Bearer <accessToken>

3. 401 응답 수신 시 (프론트 interceptor 자동 처리)
   └─► POST /api/auth/refresh { refreshToken }
         └─► 새 Access Token 수신 → 원본 요청 재시도

4. POST /api/auth/logout
   └─► 서버: Refresh Token DB에서 폐기
