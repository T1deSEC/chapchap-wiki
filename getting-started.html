Installation & Quick Start

로컬 환경에서 CHAPCHAP 전체 스택을 실행하는 방법입니다.

사전 준비

아래 소프트웨어가 미리 설치되어 있어야 합니다.

소프트웨어 버전
Docker Desktop최신
Java (JDK)22+
Node.js18+
Python3.11
Git최신

Installation

1. 레포지토리 클론

git clone https://github.com/<username>/chapchap.git
cd chapchap

2. 환경변수 설정

CHAPCHAP-Backend/ 디렉토리에 .env 파일을 생성합니다.

JWT_SECRET=dev-secret-key-at-least-32-characters-long!!
OPENAI_API_KEY=<OpenAI API 키>
OPENWEATHERMAP_API_KEY=<OpenWeatherMap API 키>
⚠ JWT_SECRET은 32자 이상이어야 합니다. 짧으면 서버 시작 시 예외가 발생합니다.

3. 데이터베이스 기동

cd CHAPCHAP-Backend
docker-compose up db -d

PostgreSQL 16 컨테이너가 :5432에서 시작됩니다. Flyway 마이그레이션은 Spring Boot 시작 시 자동 실행됩니다.

4. 백엔드 실행

.\gradlew.bat bootRun

Spring Boot가 http://localhost:8080에서 시작됩니다.

5. 프론트엔드 실행

cd CHAPCHAP-Frontend
npm install
npm run dev

Vite 개발 서버가 http://localhost:5173에서 시작됩니다. /api/* 요청은 자동으로 :8080으로 프록시됩니다.

Quick Start 스모크 테스트

모든 서비스가 정상 기동되었는지 확인합니다.

# 백엔드 직접 호출
Invoke-RestMethod http://localhost:8080/api/products

# Vite 프록시 경유 호출
Invoke-RestMethod http://localhost:5173/api/products
둘 다 { "success": true, "data": { "content": [...] } } 형태의 응답이 오면 정상입니다.

AI 서비스 추가 실행 (선택)

성분 분석 및 루틴 충돌 진단 기능에는 Python AI 서비스가 필요합니다.

단독 실행 (개발용)

cd CHAPCHAP-Backend/ai-service
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

Docker 전체 스택 실행

cd CHAPCHAP-Backend
.\gradlew.bat bootJar
docker-compose --profile full up --build -d
--profile full을 붙이면 DB + 백엔드 + AI 서비스 세 컨테이너가 모두 시작됩니다.
